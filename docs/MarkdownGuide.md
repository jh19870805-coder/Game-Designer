## Markdown 使用教学（面向你现在的文档工作）

这个文档教你在 `.md` 文件里写内容，重点围绕你现在在 `game-teardown` / `game-design` 里用得到的语法，尽量用“看到什么就照抄”的方式来讲。

---

## 一、Markdown 是什么？

- **纯文本 + 轻量标记**：就是普通文本 + 少量符号（如 `#`, `*`, `-`）来表示标题、列表、加粗等格式。
- **跨工具通用**：你在 VS Code、Cursor、GitHub、Notion 等地方看到的 Markdown 基本都是同一套规则。
- **适合做文档**：比如你的 `TeardownTemplate.md`、`TeardownRules.md`，用 Markdown 可以写得结构清晰、好维护。

> 心法：不要追求“排版完美”，只要“结构清晰 + 自己能看懂 + 以后好维护”就够了。

---

## 二、最常用的 20% 语法

### 1. 标题（用来分章节）

写法：在行首用 `#` 表示标题级别，`#` 越多级别越低。

```markdown
## 游戏概览

### 核心循环

#### 收集阶段
```

- **推荐习惯**
  - `##` 用来写一级大章节（比如“游戏概览”“核心系统拆解”）。
  - `###` 用来写二级小节。
  - 基本不用 `#`（顶级标题）和 `####` 以后，避免层级太深。

> 注意：`#` 后面**要有一个空格**，否则有些渲染器认不出来。

---

### 2. 段落与换行

- **新段落**：空一行再写，就是新的段落。
- **同一段中强制换行**：在行尾加两个空格再回车。

```markdown
这是第一段，讲整体情况。

这是第二段，中间空了一行。  
这一行和上一行属于同一段，但强制换行。
```

> 简单做法：大多数时候直接“空一行”就好，不必纠结强制换行。

---

### 3. 加粗 / 斜体（用来强调）

```markdown
**加粗的文字** 表示很重要的点。

*斜体的文字* 一般用来表示术语或需要注意的概念。
```

- **习惯建议**
  - 你现在的文档中，优先用 **加粗** 去标记“关键信息”或“小标题”。
  - 比如：`- **奖励结构**：…`、`- **目标玩家**：…`

---

### 4. 列表（你会非常常用）

#### 无序列表（点点列表）

```markdown
- **核心玩法**：种田、挂机、收集。
- **货币类型**：金币、特殊碎片。
- **平台**：PC（Steam）。
```

- 行首用 `-` + 空格，就变成一个列表项。
- 列表项内部再用 `**小标题**：说明`，会非常清晰。

#### 有序列表（编号列表）

```markdown
1. 下载并试玩游戏 2 小时。
2. 记录核心循环和资源流向。
3. 写出问题清单和设计猜想。
```

- 行首用 `1.` `2.` `3.`，记得后面跟空格。
- 实际上全写 `1.` 也可以，渲染的时候会自动编号，但为了可读性建议按顺序写。

#### 嵌套列表（列表里再分点）

```markdown
- **核心循环**
  - 收集：玩家通过挂机获得资源。
  - 花费：资源用于购买角色和建筑。
  - 强化：解锁新区域或功能，提升效率。
```

关键点：
- 子列表前面 **要缩进两个空格**（或一个 Tab），再写 `-`。

---

### 5. 引用（做备注、总结）

```markdown
> 这里的挂机收益设计看起来“无上限”，但实际被增长曲线卡住。
```

- 行首写 `>` 再空格，就表示一段引用。
- 适合写：**点评、提醒、个人猜想**。

---

### 6. 代码块 / 配置块（给自己看结构）

虽然你不写程序，也可以用“代码块”来展示结构化内容，例如一个表格结构、伪代码、配置示例等。

```markdown
```pseudo
循环：
  - 收集资源
  - 升级建筑
  - 解锁新要素
  - 回到收集资源
```
```

注意：
- 真实写的时候，外层是三个反引号 ```（键盘左上角 esc 下面那个）。
- 反引号后面可以写一个语言名（`pseudo`, `json`, `csharp` 等），只是为了高亮，随便写也没问题。
- 你在 teardown 文档里可以用这种方式来写“流程伪代码”。

> 如果你不确定这个语法，会用到时再回来抄上面的例子即可。

## 三、表格（用于结构化对比）

Markdown 表格稍微麻烦一点，但可以照抄模版用。

```markdown
| 维度         | 内容示例                          |
| ------------ | --------------------------------- |
| 目标玩家     | 喜欢轻度经营、挂机玩法的 PC 玩家 |
| 付费模式     | 一次性买断，无内购               |
| 游戏时长结构 | 短期体验 + 长期养成              |
```

规则简单记：
- 第一行：表头，用 `|` 分隔。
- 第二行：用 `-` 和 `|` 画一条分隔线（`---` 至少三个减号）。
- 后面每一行：表体内容，列数要和表头对齐。

你可以把这个当成**模板**，每次需要表格时复制一份，改里面的内容：

```markdown
| 维度     | 内容示例 |
| -------- | -------- |
|          |          |
|          |          |
```

## 四、链接与图片

### 1. 链接

```markdown
[链接文字](https://example.com)
```

在你的项目里，主要有这几种用法：
- **指向网页**：`[Steam 商店页](https://store.steampowered.com/...)`
- **指向仓库中的其他文档**：比如从 `Teardowns/README.md` 链接到某个 teardown 文档。

```markdown
[Rusty's Retirement 拆解](Teardowns/RustysRetirement/RustysRetirement_Teardown.md)
```

> 相对路径的链接：视当前 `.md` 文件所在位置来计算路径，和文件系统的相对路径一样。

---

### 2. 图片（如果你以后想插图）

```markdown
![替代文字](相对或绝对路径)
```

例子（假设你在 teardown 文档中引用 `sources/images` 下的一张图）：

```markdown
![主界面截图](sources/images/main-screen.png)
```

- `[]` 里写当图片加载不出来时显示的文字，可以简单写“主界面截图”“流程图”等。
- `()` 里写图片路径，推荐用**相对路径**，方便项目移动。

## 五、在你现有项目里的推荐写法

结合你现有的 `TeardownTemplate.md` 风格，下面是一个**推荐骨架**，以后你写新的 teardown 文档基本可以照这个结构：

```markdown
## 游戏概览

- **游戏名**：Rusty's Retirement
- **类型**：挂机 + 轻度经营
- **平台**：PC（Steam）
- **核心卖点**：XXXX

## 核心循环

- **循环概述**：一句话说明核心循环。
- **步骤**
  1. 玩家做 A
  2. 得到 B
  3. 用 B 去做 C

## 资源与数值

- **主要资源**
  - 资源 A：用途 / 来源
  - 资源 B：用途 / 来源

## 进程与解锁节奏

- **前期**
- **中期**
- **后期**

## 货币化与留存设计

> 如果是买断制游戏，可以写为什么还会让人“想回来玩”。
```

你可以复制上面的模版到任何一个新 `.md` 文件里，然后一点点填充内容。

## 六、编辑小技巧（配合 Cursor / VS Code）

- **快速插入标题**
  - 光标在行首输入 `##`，再按空格，然后输入标题文字。
- **快速生成列表**
  - 输入 `-` + 空格，开始打字；下一行按回车会自动续列表。
  - 输入两次回车，自动结束列表。
- **选中一段文字加粗**
  - 选中要强调的文字，输入 `**`，或用编辑器的快捷键（有的编辑器是 `Ctrl+B`）。
- **用“分割线”让章节更清楚**
  - 单独一行写 `---`，上下一行会被视觉上分隔开。

```markdown
上一段内容

---

下一段内容
```

---

## 七、如果你忘了语法怎么办？

很正常，不需要强行背。

- **方式 1：回到这个 `MarkdownGuide.md` 抄例子**。
- **方式 2：到 GitHub / 任何 Markdown 编辑器里，新建一个测试 `.md` 文件随便试**。
- **方式 3：直接问我 “这段想写成 XXX 效果，Markdown 应该怎么写？”**，把你想要的效果截图或描述给我就好。

> 最重要的是：**先把内容写出来，再慢慢美化结构**，不要被语法卡住你的思考。
