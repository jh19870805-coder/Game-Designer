## 2. 核心体验与目标

- **2.1 核心体验一句话总结（✅/❓）**
  - 「一款常驻桌面底部的小农场，通过自动化机器人和生物燃料循环，让玩家在处理其他事务时，被一个不断扩张、渐进解锁的农场进度条轻度陪伴。」（❓：基于实机体验与社区评价总结）
- **2.2 玩家短期 / 中期 / 长期目标（✅/❓）**
  - 短期目标（5–20 分钟尺度）（✅/❓）：
    - 建好基础农田和灌溉，开始稳定产出基础蔬菜。
    - 解锁生物燃料精炼流程，让作物→燃料→解锁新建筑形成闭环。
  - 中期目标（数小时尺度）（✅/❓）：
    - 扩展全部地块（内部记录为约 16 小时可解锁全部地块，来源：`工作表1.csv` 实机统计，✅）。
    - 解锁更多作物、自动化机器人和功能性建筑（统计中「解锁全部作物约 34 小时」，✅），提高农场自动化程度。
    - 打开动物、化石等更多子系统，完善资源循环。
  - 长期目标（多日 / 通关尺度）（✅/❓）：
    - 通过化石慢速积累与动物购买，完成动物与装饰收集（记录中「解锁全部动物 100 小时以上」，✅）。
    - 优化农场布局和自动化效率，把「桌面角落的小农场」养成高度自动、视觉丰富的长期陪伴场景。

## 3. 核心循环（Core Loop）

> 本节只描述游戏当前版本实际存在的循环。若有设计想法，放到「8. 设计参考 / 魔改想法」。

- **3.1 主循环（✅/❓）**
  - 概括：  
    - [种植蔬菜] → [等待生长 / 挂机] → [收获获得金币与素材] → [将部分蔬菜精炼为生物燃料] → [用金币 + 生物燃料解锁建筑 / 地块 / 机器人] → 回到 [更高效率的种植与收获]（✅/❓，由 `工作表1.csv` 的「种菜多样化」「大进度控制」与实机结构抽象）
- **3.2 关键驱动力（✅/❓）**
  - **解锁驱动（✅）**：  
    - 解锁新地块、新作物、新建筑、新 NPC 与动物，带来明显的视觉变化和功能升级；`工作表1.csv` 中列出多位 NPC 建筑（哈库、索内、收割者前哨、屁捏、福比克、艾克工坊、斯莱特谷仓、斯普伦克的房子）都对应新系统开启。
  - **自动化提升驱动（✅/❓）**：
    - 玩家通过购买和升级各种机器人（灌溉、收割、运输到精炼等），逐步把操作从手动变为全自动，提升「挂机效率」和「无需关注也在进步」的满足感。
  - **陪伴 / 装饰驱动（✅/❓）**：
    - 装饰物与动物系统让农场在视觉与情感层面更有陪伴感；装饰会占用部分收益地块（`工作表1.csv` 中有「装饰系统占收益地块」的记录，✅），形成在「美观 vs 效率」之间取舍的小决策。
- **3.3 主循环的输入 / 转化 / 最终产出（✅/❓）**
  - 公式抽象（✅/❓）：  
    - （输入）时间 / 注意力 + 基础作物种子 + 已有金币 / 生物燃料 →（转化）种植与生长、收割、精炼、生物燃料生成 →（产出）更多金币、生物燃料、解锁新建筑 / 地块 / 机器人 / 动物 → 回流到种植与经营循环，提升单位时间产能。
  - 输入（主循环消耗）（✅/❓）：
    - 时间（作物与浆果的生长完全依赖时间，`工作表1.csv` 中「浆果系统完全依赖时间增长」，✅）。
    - 作物种子与已有作物库存（用于继续种植与精炼，✅/❓）。
    - 金币（用于买地、购买建筑和机器人等，`工作表1.csv` 中「买地 / 盆栽土地 / 自动化机器人 / 精炼厂 / 仓库 / 水池 / 休息凳子 / 种子」均消耗货币，✅）。
    - 生物燃料（用于解锁建筑、进度关卡，`工作表1.csv` 中「建筑物条件解锁：需要生物燃料 + 货币」，✅）。
  - 转化（中间态）（✅/❓）：
    - 各类长短周期作物的生长进度、库存累积。
    - 生物燃料精炼过程中的待转化资源总量。
    - NPC 建筑与机器人升级等级（影响效率与自动化程度）。
  - 产出（主循环回报）（✅/❓）：
    - 金币：主要通过收获作物获得（`工作表1.csv` 中「货币流：收作物时获得」，✅）。
    - 生物燃料：三个不同蔬菜共同转化而成，获得数量与三种蔬菜产出生物燃料的总和相关（`工作表1.csv`，✅）。
    - 化石：由专门 NPC 缓慢挖掘获得，用于购买动物等（`工作表1.csv` 中「一个 npc 缓慢挖掘」「购买同种类动物会成倍消耗化石」，✅）。
    - 新地块 / 新建筑 / 新机器人 / 新动物解锁：提升后续生产能力与视觉反馈（✅/❓）。
  - 硬门槛 vs 软消耗（✅/❓）：
    - 硬门槛：关键建筑 / NPC 房屋 / 动物解锁往往需要特定数量的生物燃料与金币，短期内如果资金与燃料不足会直接卡住新内容（✅/❓）。
    - 软消耗：作物种植本身只消耗时间 / 少量种子，玩家可以通过更长时间挂机逐渐弥补资源不足（✅/❓）。
- **3.4 副循环（可选，✅/❓）**
  - **浆果系统（✅/❓）**：  
    - 浆果完全依赖时间增长，由机器人收割，通过蜜蜂封箱与蝴蝶加速生长，但增幅效果在当前记录中被认为不够显著（`工作表1.csv`，✅）。
  - **动物系统（✅/❓）**：
    - 不同动物对一定范围内作物的收成或生长速度提供加成（如鸡增加范围内作物收成、猪和羊提高种植速度，✅），形成围绕种植布局的「被动增益循环」。
  - **装饰系统（✅/❓）**：
    - 装饰占用产出地块（`工作表1.csv`，✅），但提供视觉与情绪价值，形成收益 vs 美观的副循环决策。
- **3.5 负循环与挫败点（✅/❓）**
  - 货币与化石方面存在一定「越想补完越吃力」的倾向：  
    - 动物购买会成倍消耗化石（✅），而化石获取由单一 NPC 缓慢挖掘（✅），导致后期若早期投资规划不佳，会感觉「化石长期短缺、补救缓慢」。
  - 自动化升级资源需求偏大（`工作表1.csv` 中「无人机后期升级使用资源量过大」「升级不能批量操作」，✅），在后期会形成「想提效却需要大量手动操作与资源」的轻度挫败点。

## 4. 主要系统模块

> 结构可按游戏实际情况增删。每个系统下，只写「原作现状」，并用 ✅ / ❓ 标注。

- **4.1 资源与货币系统（✅/❓）**
  - 资源 / 货币列表（✅/❓）：
    - 金币（✅）：收获作物时获得，是主要软通货，用于买地、购买建筑、机器人、种子等（`工作表1.csv`，✅）。
    - 生物燃料（✅）：由三种不同蔬菜精炼转化而成，数量与三者总产出生物燃料的和相关（`工作表1.csv`，✅）；是解锁建筑与大进度关卡的关键资源。
    - 作物（✅）：基础蔬菜，用于出售获得金币，也用于精炼生物燃料。
    - 浆果（✅/❓）：完全依赖时间增长，由机器人收割，是一个偏被动的长期产出子系统。
    - 化石（✅）：通过特定 NPC 缓慢挖掘获取，用于购买动物等高价值内容（`工作表1.csv`，✅）。
  - 资源流向概览（✅/❓）：
    - 作物 →（收割）→ 金币（✅）。
    - 三种蔬菜 →（精炼）→ 生物燃料（✅）。
    - 金币 + 生物燃料 →（建筑 / 机器人 / 地块购买）→ 产能与自动化等级提升（✅/❓）。
    - 化石 →（购买动物）→ 作物增益与数值加成（✅/❓）。
    - 生物燃料 ↔ 金币：`工作表1.csv` 中提到「燃料兑换比率 1:10」（燃料：货币，✅），说明二者间存在一定方向的兑换关系。

- **4.2 农场与作物 / 自动化系统（✅/❓）**
  - 农场系统（✅/❓）：
    - 地块随着进度逐步解锁；`工作表1.csv` 中记录「全部地块解锁约 16 小时」（✅）。
    - 大进度控制主要通过建筑物条件解锁（需要生物燃料 + 货币，✅），形成阶段性节点。
  - 作物解锁与多样化（✅/❓）：
    - 解锁更高级蔬菜需要当前蔬菜累计种植到一定数量（`工作表1.csv` 中「解锁下一级蔬菜需当前蔬菜种够一定数量」，✅）。
    - 生物燃料获得需要三种不同蔬菜参与（✅），鼓励多样化种植，而非只种单一高价作物。
  - 自动化设计（✅/❓）：
    - 主要角色与机器人：
      - 锈仔（Rusty）（✅）：负责灌溉和建造等基础劳动（`工作表1.csv`，✅）。
      - 灌溉机器人、收获机器人、配送到精炼生物燃料的机器人等（`工作表1.csv`，✅），逐步接管浇水、收割与运输流程。
    - 问题点（✅/❓）：
      - 后期无人机升级资源消耗过大，升级难以批量操作，被记录为设计痛点（✅）。
  - 农场规划与拆迁（✅/❓）：
    - 农场规划系统与拆迁系统在记录中被单独标为「设计问题」（`工作表1.csv`，✅）。
    - 当前只有锈仔和哈库可以完成拆迁，效率较低，且关闭游戏后搬运技能重置（✅），对后期重构布局的友好度较差。

- **4.3 解锁与进程（✅/❓）**
  - NPC / 建筑解锁（✅）：
    - 根据 `工作表1.csv`：
      - 哈库：功能与锈仔类似，参与建造 / 拆迁等。
      - 索内：开启商店，可售卖装饰。
      - 收割者前哨：出售转基因食物。
      - 屁捏：开启数据页，收集农场统计数据。
      - 福比克：开启工坊，从蜂巢收集燃料。
      - 艾克工坊：升级机器人。
      - 斯莱特谷仓：开启家畜系统，并负责挖掘化石。
      - 斯普伦克的房子：播种相关功能。
  - 阶段划分（✅/❓）：
    - 前期：以基础农田建设、作物解锁和初级自动化为主。
    - 中期：通过 NPC 建筑与工坊开出更多自动化与子系统（动物、化石、浆果等）。
    - 后期：集中在动物完善、装饰布置、高级无人机升级与整体农场优化上，时间跨度可达百小时级别（参考「解锁全部动物 100 小时以上」，✅/❓）。

- **4.4 元进程 / 长期养成（✅/❓）**
  - 动物收集与养成（✅/❓）：
    - 不同动物为作物生长 / 收成提供范围性数值加成（如鸡、猪、羊等，`工作表1.csv`，✅），需要大量化石投资。
    - 购买同种类动物的化石消耗会成倍增加（✅），形成长期缓慢推进的化石消耗坑。
  - 「全部解锁」导向的长期目标（✅/❓）：
    - 玩家有明确的「所有地块 / 作物 / 动物全解锁」路径，内部统计给出约 100 小时以上的时长预期（✅），属于典型长线养老向节奏。

- **4.5 UI / UX 特点（✅/❓）**
  - 布局与形态（✅/❓）：
    - 典型玩法是把游戏窗口贴在桌面下缘，呈长条形农场视图，便于与其他应用同时使用（❓：基于游戏卖点与社区截图）。
  - 操作成本与信息呈现（✅/❓）：
    - 主要操作集中在单屏农场与少量菜单中，系统数量较多但分布在不同 NPC 房屋与面板里。
    - 自动化与优先级调度在记录中被多次讨论，说明 UI 对「批量升级 / 批量调整」支持有限，会放大后期操作负担（`工作表1.csv` 的问题记录 + 社区反馈，✅/❓）。

## 5. 核心数值与节奏（可选，✅/❓）

- **5.1 关键倍率与增长曲线（✅/❓）**
  - 时间线统计（来源：`工作表1.csv`，✅/❓）：
    - 全部地块解锁：约 16 小时。
    - 解锁全部作物：约 34 小时。
    - 解锁全部动物：100 小时以上。
  - 从这些节点可以推断：  
    - 前期解锁较快，中期向后期过渡时，动物与高阶自动化的资源需求呈显著抬升，属于「中后期拉长线」的增长曲线（❓）。

- **5.2 时间结构（✅/❓）**
  - 作物与浆果生长高度依赖实时经过（✅），非常适合当作「后台进度条」存在。
  - 玩家可以短时上线处理布局与升级，然后长时间挂机等待资源累积，这是典型的「主动规划 + 长时放置」时间结构（✅/❓）。

## 6. 内容广度与重复度（✅/❓）

- **6.1 内容品类（✅/❓）**
  - 作物：多档解锁梯度，通过累计种植数量解锁更高级作物（✅/❓）。
  - 建筑与功能房屋：多位 NPC 与功能建筑（商店、工坊、数据屋、动物谷仓等）逐步开放（`工作表1.csv`，✅）。
  - 动物：多种动物对农场提供不同增益，且购买成本与化石投入逐级提高（✅）。
  - 机器人：多类机器人分别负责浇水、收割、运输、燃料收集等（✅/❓）。
- **6.2 重复度与变体（✅/❓）**
  - 主行为（种植→等待→收获→解锁）高度重复，但通过作物种类、建筑与自动化层级的变化提供一定变体（✅/❓）。
  - 记录中提到「浆果系统完全依赖时间增长」「自动化升级成本高且难以批量操作」等问题（✅），说明部分子系统在后期会放大重复劳动感。

## 7. 亮点与问题（备注区，可分散写）

- **7.1 本作亮点（✅/❓）**
  - 「桌面角落农场」的形态与定位比较独特：强调低占用、长陪伴，让玩家在做其他事时也能感受到农场在持续生长（✅/❓）。
  - 农场、动物、机器人、化石等多个子系统围绕同一块小农场叠加，形成较丰富的长期成长线（✅/❓）。
  - 资源系统中「三种蔬菜合成生物燃料」「化石用于高价值动物」「燃料与货币存在兑换关系」等设计，让主循环在结构上有一定深度（`工作表1.csv`，✅/❓）。

- **7.2 已观察到的问题点（✅/❓）**
  - 自动化后期数值需求过高、升级操作不能批量处理（`工作表1.csv`，✅），在一个主打「放置与解压」的游戏中增加了不必要的操作负担。
  - 拆迁系统效率低，仅由个别角色负责，且关闭游戏后搬运技能会重置（✅），对喜欢经常重构布局的玩家不够友好。
  - 浆果系统虽然有蜜蜂 / 蝴蝶加速，但增幅感知不强（`工作表1.csv`，✅），在资源结构中存在感偏弱。
