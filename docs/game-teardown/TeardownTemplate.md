## 3. 核心循环（Core Loop）

> 本节只描述游戏当前版本实际存在的循环。若有设计想法，放到「8. 设计参考 / 魔改想法」。

- **3.1 主循环（✅/❓）**
  - 用 1–3 行描述玩家在典型一段游戏时间里的反复行为序列。  
  - 示例结构（按实际游戏改写）：  
    - [采集 / 种植] → [等待 / 挂机] → [收获资源] → [解锁/升级] → 回到 [采集 / 种植]。
- **3.2 关键驱动力（✅/❓）**
  - 驱动力 = 玩家愿意重复主循环的“理由”。通常来自：成长变强、内容解锁、收集/完成、叙事推进、风险收益、社交比较、随机惊喜、外部奖励等。
  - 建议写 1–3 条最主要驱动力，并各自补一句“它通过什么反馈让玩家感受到”（例如：数字跳动、掉落音效、解锁动画、剧情节点、排行榜名次变化）。
- **3.3 主循环的输入 / 转化 / 最终产出（✅/❓）**
  - 目的：把 **3.1 主循环**落到“资源账本”，看清循环靠什么启动、靠什么维持、回报落在哪。
  - 先写一句公式（按实际游戏改写）：  
    - （输入）时间/精力/操作 + 资源 A（来自哪里） →（转化）玩法动作/系统处理 →（产出）资源 B/解锁 C/战力 D → 回流到主循环的哪一步
  - 再列清单把资源对上（尽量对应「4.1 资源与货币系统」里的命名）：  
    - 输入（主循环消耗）：[体力/行动点]、[软通货]、[材料]、[时间]、[注意力]……
    - 转化（中间态）：[半成品/进度条/建设度/经验]……
    - 产出（主循环回报）：[硬通货]、[新区域/新工具解锁]、[产能提升]、[叙事推进]……
  - 标注哪些是“硬门槛”（卡进度的）与“软消耗”（多花点时间就能补回的）。
- **3.4 副循环（可选，✅/❓）**
  - 副循环 = 附着在主循环旁边的“侧支循环”，不一定每次都跑，但会提供加速/增效/变现/换玩法节奏（例如：制作→交易、任务→领奖、抽卡→养成、基地建设→长期增益）。
  - 每个副循环用一行写清：  
    - 触发条件 → 消耗了主循环的哪些资源（来自 3.3 的输入/产出）→ 产出了什么 → 回流到主循环的哪一步（加速/解锁/增强/换取）
- **3.5 负循环与挫败点（✅/❓）**
  - 负循环 = “越失败越难翻盘 / 越玩越弱”的恶化闭环（注意：这不是副循环）。例如：资源破产→无法修复/补给→更易失败→进一步破产。
  - 如果基本没有明显负循环，也写结论：失败主要是“慢一会儿/少拿点”的软惩罚，局面不会持续恶化。

## 4. 主要系统模块

> 结构可按游戏实际情况增删。每个系统下，只写「原作现状」，并用 ✅ / ❓ 标注。

- **4.1 资源与货币系统（✅/❓）**
  - 资源/货币列表：  
    - 例：Scrap（备件）（✅）：主要用途……  
    - 例：Biofuel（生物燃料）（✅）：主要用途……  
  - 资源流向概览：谁产出、谁消耗、闭环/开环情况。

- **4.2 核心生产/战斗/经营系统（✅/❓）**
  - 按实际游戏改名，例如「农场与作物系统」「战斗与关卡系统」等。
  - 包括：对象种类（作物/敌人/建筑等）、关键属性字段、交互方式、反馈表现。

- **4.3 解锁与进程（✅/❓）**
  - 解锁方式（数值门槛 / 关卡进度 / 任务等）。
  - 是否有明显的阶段划分（前期 / 中期 / 后期）及其特点。

- **4.4 元进程 / 长期养成（✅/❓）**
  - 如：天赋树、账号等级、周目系统、继承要素等。

- **4.5 UI / UX 特点（✅/❓）**
  - 主要布局、信息密度、操作成本（需要切多少层菜单）、可视化手法等。

> 视具体游戏补充更多子系统小节，例如「任务与事件」「社交与多人」「番茄钟 / 外部工具集成」等。
